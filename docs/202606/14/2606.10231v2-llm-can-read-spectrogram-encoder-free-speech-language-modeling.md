## 摘要
最近的语音感知大语言模型（Speech-LLMs）依赖预训练的语音编码器将音频转换为大语言模型可用的语义丰富表示。在这项工作中，我们反而探索：大语言模型能否直接学习读取梅尔语谱图，而无需专用的语音编码器？我们提出Mel-LLM，一种无编码器的语音大语言模型，通过线性投影将经过轻度预处理的梅尔语谱图块直接输入到大语言模型中，使大语言模型仅通过自身参数学习语音-文本对齐。我们在自动语音识别（ASR）和文本到语音（TTS）任务上进行了大量实验。在ASR方面，我们在OpenASR排行榜公共集和生产级扩展实验上进行了评估，结果表明无编码器方案在性能上具有竞争力，与使用编码器初始化的方案相比仅有有限退化。我们发现，当数据有限时，从多模态检查点（Phi-4-MM）初始化对于保持性能至关重要。我们还进行了消融研究，揭示了哪些大语言模型层与语音编码相关性较低。在TTS方面，我们通过下一标记变分自编码器方法展示了初步结果。虽然TTS性能尚未达到最优，但这些结果表明了用于自回归语音-文本建模的完全统一无编码器架构的可行性。

## Abstract
Recent speech-aware large language models (Speech-LLMs) rely on a pre-trained speech encoder to convert audio into semantic-rich representations consumable by LLM. In this work, instead, we explore: can an LLM learn to read Mel spectrogram directly without a dedicated speech encoder? We propose Mel-LLM, an encoder-free Speech-LLM that feeds lightly pre-processed Mel spectrogram patches directly into the LLM through a linear projection, allowing the LLM to learn speech-text alignment purely through its own parameters. We conduct extensive experiments on both automatic speech recognition (ASR) and text-to-speech (TTS) tasks. For ASR, we evaluate on the OpenASR leaderboard public sets and production-level scaling experiments, demonstrating that the encoder-free solution achieves competitive performance with only limited degradation compared to encoder-initialized counterparts. We find that when data is limited, initialization from a multimodal checkpoint (Phi-4-MM) is crucial for maintaining performance. We also present ablation studies revealing which LLM layers are less relevant to speech encoding. For TTS, we show preliminary results with a next-token VAE approach. While TTS performance is not yet optimal, these results establish the feasibility of a fully unified encoder-free architecture for autoregressive speech-text modeling.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：现有语音大语言模型（Speech-LLM）普遍采用“语音编码器 → 模态投影器 → 大语言模型”的三段式架构。编码器（如 Whisper、Conformer）体积庞大（如 Whisper-large 超 6 亿参数），带来计算开销、表示不匹配、信息瓶颈等问题。
- **核心问题**：大语言模型是否可以不依赖专用语音编码器，直接读取原始梅尔频谱图（Mel spectrogram）并完成语音-文本理解与生成？
- **整体含义**：如果可行，则能大幅度简化模型架构、降低参数量、避免编码器-LLM 之间的表示不匹配，并推动统一的语音-文本联合建模。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **Mel-LLM**：移除传统语音编码器（特别是 Transformer/Conformer 块），仅保留轻量卷积层用于下采样（可选），将梅尔频谱图分块后经线性投影直接送入大语言模型，使 LLM 自身学习隐式地承担“语音编码”功能。

### 关键技术细节

#### ASR 路径（图 1a）
1. **特征提取**：80 维对数梅尔频谱图，后接均值方差归一化（MVN）。
2. **轻量卷积下采样**：时间轴压缩因子 r（默认 r=8，对应 12.5 Hz 帧率），卷积层可选，r=1 时禁用。
3. **线性投影**：单个线性层将特征映射到 LLM 隐藏维度（d_LLM=3072）。
4. **LLM 处理**：Phi-4-MM（32 层，3072 维），LoRA 微调（rank=320, α=640），仅训练 LoRA 参数，基座模型冻结。
5. **输出**：LLM 自回归生成文本 token。

#### TTS 路径（图 1b）
1. **输入**：文本 token 序列。
2. **LLM 生成**：自回归产生 LLM 隐状态（对应每帧梅尔频谱图位置）。
3. **VAE 解码器（Mel Head）**：
   - 隐状态 → 线性层 → 均值和方差（μ, log σ²）→ 重参数化 → 残差 MLP（3 层 tanh）→ 初步梅尔帧。
   - 后处理：5 层 Conv1D 残差 Postnet 精炼。
   - 停止预测：线性层 + 二分类交叉熵，预测语音结束。
4. **训练损失**：L1+MSE 重构损失 + KL 散度 + 停止损失 + 通量损失（时间平滑），同 MELLE [14]。
5. **波形重建**：使用 HiFi-GAN 声码器。

## 6. 论文的主要结论与发现

1. **无编码器 ASR 可行且具竞争力**：Mel-LLM 在 OpenASR 上平均 WER 7.12%，仅比使用随机编码器初始化方案低 0.15%（6.97%）。
2. **多模态预训练初始化在数据有限时至关重要**：随机初始化 LLM 则降至 7.44%，而 Phi-4-MM 初始化带来显著提升。
3. **数据缩放可弥合差距**：在 10× 内部数据下，无编码器方案与编码器方案差距从 +11.3% 缩小至 +3.8%。
4. **下采样率影响效率与性能**：12.5 Hz 速度比编码器快 1.57×，且 WER 仅略高（7.12% vs 6.97%）。
5. **LLM 下层负责隐藏的语音编码**：冻结 LLM 上层（L24-31）几乎不影响性能（7.43% vs 7.12%），说明上层已具备高级语义能力，下层需适应语音特征。
6. **TTS 可行性初步证实**：Phi-4-MM 初始化 + VAE 解码可实现可听合成，但 WER 较高（11.03%~14.75%），未达 SOTA。

## 8. 不足与局限
