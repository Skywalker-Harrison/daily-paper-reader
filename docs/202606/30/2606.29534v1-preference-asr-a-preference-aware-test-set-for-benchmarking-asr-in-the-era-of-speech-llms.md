## 摘要
流行的ASR测试集在数字、非流利、实体和大小写上采用不一致的约定，而标准归一化器会抹去用户关心的格式差异。因此，当前基准无法衡量模型是否遵循用户对输出风格的偏好。我们引入Preference-ASR，一个评估ASR系统在四个类别（归一化、实体、非流利和大小写）中遵循自然语言偏好指令能力的测试集。该测试集通过两阶段LLM辅助流程（含人工验证）从七个开源语料库构建，并使用偏好感知归一化器进行评估，该归一化器有选择地跳过与当前指令匹配的步骤。对四个模型的基准测试显示，排名在不同偏好类型间发生变化，揭示了传统评估所掩盖的质量差异。我们公开发布该数据集。

## Abstract
Popular ASR test sets adopt inconsistent conventions for numbers, disfluencies, entities, and casing, while standard normalizers erase the format distinctions users care about. Current benchmarks therefore cannot measure whether a model follows user preferences for output style. We introduce PreferenceASR, a test set evaluating ASR systems on their ability to follow natural-language preference instructions across four categories: normalization, entities, disfluencies, and case. Built from seven open-source corpora via a two-stage LLM-assisted pipeline with human verification, it is evaluated with a preference-aware normalizer that selectively skips steps matching the active instruction. Benchmarking four models shows rankings shift across preference types, exposing quality differences traditional evaluation obscures. We publicly release the dataset.

# 论文详细总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前主流的自动语音识别（ASR）基准测试集（如 LibriSpeech、GigaSpeech、Earnings-22、AMI 等）在转录格式约定上存在严重不一致。例如，数字有的采用口语形式（如“seventy percent”），有的采用书面形式（如“10%”）；对于不流畅的说话（如填充词、重复）有的保留、有的删除；大小写和标点的处理也各不相同。这种混乱导致模型的排名往往只反映它“恰好匹配”了哪个数据集的注释风格，而非真实转录质量。
- **背景与动机**：随着 Speech LLMs 的出现（如 SALMONN、Qwen-Audio、Qwen2.5-Omni），它们能够理解自然语言指令，并根据指令调整输出风格。然而，现有的评估体系仍采用固定的归一化（lowercase、数字转单词、去标点）来计算词错误率（WER），这恰恰抹除了用户关心的格式差异（例如，用户可能要求输出数字（ITN）或单词形式（TN），保留或不保留不流畅词等）。因此，迫切需要一个新的基准来评估 ASR 系统的**指令跟随能力**，即能否按照用户偏好指令（preference instruction）输出指定格式的转录。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建一个**偏好感知（Preference-Aware）的测试集**，包含音频-指令-参考转录三元组，并设计一个**偏好感知的归一化器**（preference-aware normalizer），在计算 WER 时根据指令有选择地跳过某些归一化步骤，从而公平评估模型是否真正遵循了偏好指令。
- **关键技术细节**：
  - **偏好类别定义**：共四大类，每类含子类别：
    - **归一化（Normalization）**：数字（ITN 输出“22” vs TN 输出“twenty-two”）、符号（“$100” vs “one hundred dollars”）、网址链接（“www.google.com” vs “www dot google dot com”）。
    - **实体（Entities）**：公司、产品、人名等，还包括不在音频中的“假阳性”实体（false positives），用于测试模型是否产生幻觉。
    - **不流畅（Disfluencies）**：填充词（uh、um）、重复（I I think）、假起始（I was go- going）、错误语法。
    - **大小写（Case）**：全小写无标点 vs 正确大写和标点。
  - **数据集构建流程（两阶段***）：
    - **阶段一：偏好分类**。使用 Qwen3-30B-A3B 模型对每个样本的转录进行偏好类别标注（如某样本包含数字和“um”，则标为归一化和不流畅）。
    - **阶段二：指令与参考生成**。对于每个类别，为样本生成一个或两个指令（如“将数字和符号归一化为口语形式”），并生成对应的“偏好文本”（preference text）。采用优先级去重：归一化 > 实体 > 不流畅 > 大小写。
    - 全程包含 **人工验证与修正**：对初始转录进行人工校订，并对生成结果进行审核。
  - **偏好感知归一化器**：在标准 WER 计算前，根据指令类型选择性关闭对应步骤：
    - 对于**归一化偏好**：若指令要求 ITN，则跳过数字→单词转换；若要求 TN，则跳过单词→数字转换。
    - 对于**不流畅偏好**：若指令要求保留，则保留重复和填充词；若要求移除，则从参考和假设中一起移除。
    - 对于**大小写偏好**：不对大小写和标点进行归一化。
    - 对于**实体**：保持标准归一化，因为 WER 自然反映实体名称准确性。

## 3. 实验设计：数据集、基准、对比方法

- **数据集**：Preference-ASR 测试集，共 3,210 个三元组，来自 7 个开源语料库：AMI、Common Voice、Earnings-22、GigaSpeech、LibriSpeech、SPGISpeech、VoxPopuli。分布见表1。每个样本附带偏好类别和具体指令。
- **基准**：标准 WER（使用完全归一化）与偏好感知 WER（有选择地跳过匹配步骤）两套评价指标。
- **对比方法**：
  - **Parakeet-TDT-0.6B-v3**：传统非 LLM 基线（FastConformer 编码器 + TDT 解码器，无指令输入）。
  - **Canary-Qwen-2.5B**：FastConformer 编码器 + Qwen2.5 LLM 后端（LoRA 适配），但训练时无偏好微调。
  - **Phi-4-Multimodal (5.6B)**：原生支持指令的多模态模型。
  - **Qwen3-Omni-30B**：最强的指令感知系统，训练中使用了上下文偏置。

## 6. 论文的主要结论与发现

- **模型排名随偏好类型显著变化**，传统 WER 隐藏了这种差异。例如：
  - Qwen3-Omni-30B 在归一化、不流畅和大小写指令上表现出良好的遵从性（偏好感知 WER 低于 Canary 和 Parakeet），但在实体指令中发生了严重的幻觉（WER 从 5.12% 飙升至 12.85%），说明模型被文字提示所误导，无法坚持声学证据。
  - Canary-Qwen-2.5B 对指令几乎无响应（WER 变化极小），说明仅靠 LLM 后端而无偏好对齐训练不足以实现指令跟随。
  - Phi-4-Multimodal 在不流畅指令上表现异常：默认状态下严重去除不流畅（WER 50.18%），但在明确指令下能大幅降低（10.46%）；但在大小写指令上却急剧恶化（3.93% → 19.76%），说明指令跟随是类别相关的。
- **偏好感知归一化器暴露了模型在格式上的偏差**：在未给出偏好指令时（默认提示），模型的输出格式已经倾向于某种默认选择（如 Canary 在大小写上 WER 从 3.59% 跳到 10.08%），说明其输出格式与测试集目标不一致。
- **提示措辞的敏感性**：即使是微小的指令措辞变化也会影响 WER（如 Phi-4 在标准样本上默认与指令版本的 WER 从 4.41% 变到 3.88%），展现了 Speech LLMs 对提示文本的敏感度。

## 7. 优点

- **问题导向**: 精准抓住了现有 ASR 评估的核心缺陷（格式约定不一致、用户偏好无法表达），提出实际亟需的测评方向。
- **方法创新**: 提出偏好感知归一化器，简单但有效，避免了传统 WER 对指令遵循的误判。
- **构建流程严谨**: 两阶段 LLM 辅助 + 人工验证，保证了数据集质量；优先级去重确保每个样本只测试一类偏好。
- **公开性**: 明确开源数据集和评估代码，促进可重复性研究和社区发展。
- **丰富分析**: 不仅报告总体 WER，还按类别、按数据集细分，并分析了幻觉、模型默认行为等深层问题。

## 8. 不足与局限

- **语言覆盖**: 仅限于英语，未扩展到多语言场景。
- **偏好类别**: 当前只包含四大类别，未来可增加多说话人偏好、话音风格偏好等。
- **人工依赖**: LLM 生成的偏好文本需要人工验证，尤其归一化类别中声学上下文无法从文本推断，仍需大量人力。
- **模型范围**: 仅测试了 4 个模型，缺乏对更多开源/商业 Speech LLMs（如 Whisper 系列、Gemini 等）的比较。
- **幻觉测试**: 实体类别中仅包含“假阳性”实体，未测试其他潜在幻觉形式（如凭空生成数字、日期等）。
- **无训练**: 未测试对模型进行偏好对齐训练后的效果（论文只评估了现有模型的指令跟随能力）。
- **数据集规模**: 3,210 个三元组相对适中，但不足以完全覆盖所有真实场景中的偏好指令组合。
