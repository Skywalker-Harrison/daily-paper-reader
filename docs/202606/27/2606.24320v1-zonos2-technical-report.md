## 摘要
我们提出了 ZONOS2 8B，这是我们最新的 TTS 模型，在自然度、韵律和语音克隆保真度上达到了业界领先水平。我们在规模、数据和训练方案上对 Zonos-v0.1 进行了改进。我们通过一种新颖的混合专家（MoE）主干网络将模型从 1.6B 参数扩展到 8B 参数（其中 900M 活跃），从而提升了推理延迟和吞吐量。我们使用新的数据处理流水线将训练语料从 20 万小时扩展到超过 600 万小时，并简化了后训练和条件化方案以提升自然度和语音克隆保真度。我们在质量、说话人相似度、WER 以及我们新提出的 TTS 基准 ZTTS1-Eval 上对 ZONOS2 8B 进行了评估，结果显示其与业界领先系统竞争力相当，同时保持良好的流式延迟。我们在 GitHub 和 Hugging Face 上以 Apache 2.0 许可证发布了模型权重和示例推理代码。

## Abstract
We present ZONOS2 8B, our latest TTS model, which achieves state-of-the-art naturalness, prosody, and voice cloning fidelity. We improve upon Zonos-v0.1 across scale, data, and training recipe. We scale the model from 1.6B to 8B total parameters (900M active) with a novel mixture-of-experts (MoE) backbone, improving inference latency and throughput. We expand our training corpus from 200K to over 6M hours using a new data processing pipeline, and we simplify our post-training and conditioning recipes to improve naturalness and voice cloning fidelity. We evaluate ZONOS2 8B on quality, speaker similarity, WER, and ZTTS1-Eval, our novel TTS benchmark, where it performs competitively with state-of-the-art systems while maintaining good streaming latency. We release our model weights and example inference code under an Apache 2.0 license on GitHub and Hugging Face.

## 论文总结：ZONOS2 8B 技术报告

### 1. 论文的核心问题与整体含义（研究动机和背景）
- **研究动机**：现有端到端 TTS 模型在自然度、韵律和声音克隆保真度上仍有提升空间，且往往只优化某一维度（如高质量但可控性差，或高效但语言支持少）。ZONOS2 旨在设计一个全方面强大的 TTS 系统，特别关注自然度和语音克隆保真度，同时保持低延迟推理。
- **贡献**：发布一个基于混合专家（MoE）架构的 8B 参数（900M 活跃）开源 TTS 模型，训练语料从 200K 小时扩展到 6M+ 小时，并引入新的训练后处理和条件配方，同时提出一个新的 TTS 评估基准 ZTTS1-Eval 以弥补现有基准的不足（语言覆盖少、评分模型陈旧、缺乏韵律和多样性度量）。

### 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：使用解码器‑仅 MoE Transformer 架构，将文本（字节级分词）和音频（RVQ 编解码器）作为输入，以自回归方式预测音频码本 token。
- **关键技术细节**：
    - **音频分词**：使用 Descript Audio Codec (DAC)，9 个码本，通过延迟模式（delay pattern）生成：对音频帧 t 和码本 j，token Y[t,j] = X[t-j, j]（当 t>=j），否则为 padding。解码时逆运算，并行预测多个码本，降低流式解码延迟。
    - **文本分词**：采用 UTF-8 字节级分词，无需语言特定的 G2P 音素转换，避免预处理错误，提升多语言和代码混合场景的鲁棒性。
    - **说话人嵌入**：使用 ECAPA‑TDNN 提取 2048 维嵌入，再通过 LDA 投影到 1024 维以降低信息泄露（如时长、背景噪声、词汇内容），然后通过投影层加到序列前缀。
    - **说话速率条件化**：将速率量化为离散桶，作为 token 插入文本前。
    - **质量条件化**：
        - 在训练时对克隆音频进行增强（噪声、音乐、压缩、混响），使模型对低质量克隆更鲁棒，同时添加“增强嵌入” token。
        - 引入合成文本 token 表示生成带宽、音量、前后静音帧、信噪比等。
        - 在退火阶段引入“Quality Mode” token，允许用户选择高质量（高可懂度但可能牺牲克隆保真度）。
    - **MoE Transformer 架构**：28 层，隐藏维度 2048，前 3 层和最后一层为密集层，其余为 MoE 层（16 个专家，除最后 MoE 层 top-2 外均为 top-1 路由），使用分组查询注意力（GQA, 16 查询头 / 4 键值头）、RoPE 位置编码、Qwen 门控（headwise）、路由器使用 EDA 和三层 MLP。
    - **训练目标**：标准的自回归条件语言模型损失，对非填充音频码本计算负对数似然，加上路由器平衡辅助损失（基于零均值偏置向量）。
    - **训练阶段**：
        - 预训练（77,500 步，2.9T tokens）无说话人条件。
        - 中训练（15,000 步，560B tokens）提高 ASR 转录一致性过滤。
        - 第一阶段退火（10,000 步）引入说话人嵌入（从随机裁剪的目标音频计算，掩盖该段损失）和条件 token。
        - 第二阶段退火（10,000 步）说话人嵌入覆盖整个目标序列，去掉损失掩盖，引入 Quality Mode token。

- **训练数据**：6.2M 小时，包括公开语音语料、播客、有声书、对话数据集、多语言网络语音数据、表达性/角色化语音数据。数据加载可按子数据集上采样。

    - **闭源**：Cartesia Sonic 3.5、Eleven Labs V3、Gemini 3.1 Flash、Inworld TTS 2。

### 6. 论文的主要结论与发现
- ZONOS2 8B 在自然度、韵律和声音克隆上达到或超越现有 SOTA，尤其在 ITW 集上的 TTSDS2 韵律得分领先。
- Quality Mode 能显著降低 WER 并提升 UTMOS 但会轻微降低说话人相似度（在 Clean 集上部分语言有提升，ITW 集上普遍降低）。
- ZONOS2 在生成多样性（DS‑WED）上明显优于所有对比模型，在韵律分布（Allosaurus SR 距离）上更接近真实语音。
- 发现 MoE 在音频数据上的路由平衡比在文本数据上更困难，推测与 DAC 延迟 token 的固有难度有关。
- 字节级分词比音素级分词更有益于多语言和鲁棒性，避免了 G2P 失败的静默错误。
- 说话人嵌入的 LDA 投影和两阶段训练被证明对于避免信息泄露和过拟合至关重要。

### 7. 优点
- **技术创新**：首次将 MoE 引入开源 TTS，实现参数扩展同时保持低推理延迟；通过 LDA 投影有效缓解说话人嵌入的信息泄露问题。
- **数据与训练**：大规模数据处理流水线（6.2M 小时，多语言多来源），ASR 集成转录过滤，条件化训练（速率、质量、增强）提升可控性和鲁棒性。
- **基准贡献**：提出 ZTTS1‑Eval，覆盖更多语言、加入自发语音、使用最新评分模型、增加韵律和多样性维度，弥补了现有基准的不足。
- **开放发布**：模型权重和推理代码以 Apache 2.0 开源，促进研究与应用。

### 8. 不足与局限
- **稳定性问题**：MoE 训练在音频数据上不稳定（路由器熵可能坍塌），需手动调整密集层和 top‑2 路由，未来需进一步探究原因和解决方案。
- **性能权衡**：Quality Mode 虽提升可懂度和质量但降低说话人相似度，用户需根据场景取舍。
- **推理与约束**：使用 GQA（而非更稳定的 MHA）是为了速度牺牲了一些质量，且延迟模式需要 N‑1 帧的前瞻，可能影响极低延迟场景。
- **多语言覆盖不均衡**：如泰语（th）、他加禄语（tl）等低资源语言 WER 仍然较高（即使 Quality Mode 下）。
- **说话人嵌入仍有泄露风险**：即使使用 LDA，仍需两阶段训练，否则仍会出现“glossolalia”或无声输出，说明完全解耦尚不完美。
- **计算资源未公开**：未说明 GPU 型号、数量和训练时间，影响复现和经济性评估。
- **实验局限性**：未进行系统性的消融实验（如不同编解码器、不同条件策略的对比），也未在更多闭源商业系统上测试（如 ElevenLabs 和 Gemini 不支持真正的零样本克隆，直接对比欠公平）。
