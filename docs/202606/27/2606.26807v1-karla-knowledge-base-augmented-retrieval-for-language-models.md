## 摘要
我们提出了一种新方法，使得大型语言模型（LLM）在生成令牌时能够自动从知识库中获取事实知识。这意味着：（1）LLM输出中的事实知识可以在不重新训练LLM的情况下更新；（2）LLM输出中的事实可以追溯到知识库，从而实现透明性和可解释性；（3）较小的模型可以与较大的模型达到相同的事实准确性。我们的核心思想是训练模型生成特殊的令牌，以触发对知识库的查询。我们的实验表明，我们的方法在短文本和长文本生成中均提高了事实基础性，并且允许通过知识库编辑而非参数更新来实现事实修订。

## Abstract
We propose a new method that allows an LLM to automatically pull in factual knowledge from a knowledge base during token generation. This means that (1)~factual knowledge in the LLM output can be updated without retraining the LLM, (2)~facts in the LLM output can be traced to the knowledge base for transparency and explainability, and (3)~smaller models can achieve the same factual accuracy as larger models. Our core idea is to train the model to produce special tokens that trigger a query to the knowledge base. Our experiments show that our method improves factual grounding in both short and long-form generation, and allows factual revisions to take effect through KB edits rather than parameter updates.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）在预训练中隐式存储事实知识，导致事实难以检查、修正或选择性更新。现有问题包括：幻觉、知识随时间过时（时间错位）、对长尾实体知识回忆不均、依赖参数规模存储知识效率低下。
- **研究动机**：分离语言能力与事实知识存储，让 LLM 从外部知识库（KB）中可靠获取事实，从而实现无重训的知识更新、事实可追溯、小模型高事实准确性。
- **整体含义**：KARLA 是一种后训练方法，使模型在生成令牌时自动触发 KB 查询，将事实知识外化到 KB 中，通过 KB 编辑即可更新模型行为，无需大规模参数更新。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 训练 LLM 生成特殊的“关系令牌”`⟨r⟩`，后接主体标记`⟨subj⟩...⟨/subj⟩`，触发对 KB 的查询。查询结果（对象 o）以`⟨KB⟩o: o_desc⟨/KB⟩`格式插入生成序列，供后续令牌条件化。
- 通过合成训练数据（由教师 LLM 从 KB 生成百科全书式段落，并嵌入查询标记）微调模型，同时采用**遮蔽下一令牌预测损失**：对 KB 返回的对象范围设置损失掩码 `m_t=0`，防止模型记忆事实，鼓励依赖检索。

### 关键技术细节
1. **谓词表示**：每个关系用一个原子特殊令牌`⟨r⟩`表示，嵌入初始化为对应子词嵌入均值；避免模型生成不存在的谓词。
2. **训练数据生成**：
   - 采用**谓词平衡采样**（Algorithm 1）：贪心选择当前出现最少的关系，从该关系实体集中均匀采样一个实体，再从其邻域中选取至多 k 个事实（其他关系），使每个关系出现次数介于 T 和 2T 之间。
   - 使用教师 LLM（GPT-5 mini）根据实体子图和关系描述生成约 250 令牌的段落，并标注查询位置；通过结构化反馈验证（对象文本精确匹配、所有采样关系均出现），失败重试最多 2 次。
3. **训练**：
   - 基于 Qwen3-Base，LoRA 微调（rank=64, α=128），训练新谓词令牌嵌入和输出投影。
   - 损失函数：`L(θ) = -∑ m_t · log p(˜x_t | ˜x_<t)`，其中 `m_t=0` 对 KB 返回对象范围，`m_t=1` 对其他令牌。
   - 10% 的查询随机替换为失败令牌 `⟨KB_FAIL⟩`，以处理 KB 无匹配情况。
4. **推理**：
   - 预处理：将提示中的实体 mention 通过两阶段实体链接（bi-encoder 候选检索 + 小模型 rerank）解析为 KB 规范实体，并包装为`⟨KB⟩e*: e*_desc⟨/KB⟩`。
   - 自回归生成时，一旦检测到`⟨r⟩⟨subj⟩s⟨/subj⟩`序列，暂停生成，解析 s 为规范实体，查询 KB 返回对象或失败令牌，恢复生成。

### 算法流程（文字说明）
1. 谓词平衡采样：初始化计数，循环选择最少出现的关系 → 采样实体 → 选取该实体下最多 k-1 个其他关系 → 记录事实 → 更新计数，直到所有关系出现次数≥T。
2. 教师 LLM 根据实体子图生成带查询标记的段落。
3. 微调模型：输入带查询标记的序列，使用遮蔽损失训练。
4. 推理：输入提示 → 实体链接 → 自回归生成 → 遇特殊令牌触发 KB 查询 → 插入结果继续生成。

## 3. 实验设计

### Benchmark 和对比方法

## 6. 主要结论与发现

1. **KARLA 有效降低目标归一化困惑度**：在 YAGO 和 PrimeKG 上，KARLA 比参数化 SFT 相对降低 16%-21% 的困惑度，表明外部证据带来的不确定性减少超过了查询令牌的代价。
2. **KARLA 提升事实准确性**：在 PopQA 上，KARLA（Qwen4B）达到 80.91%，远超基线（23.41%）和 1-hop RAG（56.17%）；在 FActScore 上同样领先（58.9% vs 24.16% 基线和 56.8% RAG）。小模型 KARLA（0.6B）即可超越大模型基线和 LMLM。
3. **KARLA 支持即时知识更新**：反事实实验中，仅替换 KB 即可达到 96.1% 准确率，无需任何梯度步；而参数化 SFT 需要数百步；1-hop RAG 仅 87.3%，且对流行实体（Q4）准确率下降至 77.8%（模型置信度强时忽略上下文）。
4. **内联查询预测准确率高**：主语准确率 >99%，关系准确率 87-98%，联合准确率高。
5. **消融证实关键组件有效**：特殊令牌优于自由形式（消除 0% 架构外谓词发射）；实体描述对长尾问答至关重要；空 KB 使性能下降至基线水平，确认收益来自 KB 查询。

## 7. 优点

- **新颖分离范式**：通过训练模型发出查询令牌，将事实知识外化到 KB，同时保留语言能力。比 RAG 更严格地归因（每个 KB 返回的文本可直接追溯）。
- **合成数据生成方法**：谓词平衡采样 + 教师 LLM 验证，确保训练覆盖均匀且质量高，避免自然语料中关系分布不均和提取错误。
- **资源高效**：小模型（0.6B/1.7B）即可达到甚至超越大模型的事实准确性，且更新只需编辑 KB，无需重训。
- **实验设计全面**：覆盖短问答、长文本生成、知识更新场景，多项消融验证，与多个相关基线公平对比，包括 LMLM 外部记忆方法。
- **可扩展性**：知识库大小不影响模型参数规模，仅关系数量影响。适用于百万级事实的大型 KB（如 YAGO）。

## 8. 不足与局限

- **谓词词汇封闭**：模型只能查询训练中见过的谓词，扩展新关系需重新训练。无法处理未见关系。
- **实体消歧依赖**：需要高效准确的实体链接，这是开放问题；虽然论文中使用了两阶段方法，但在真实部署中仍有挑战。作者提到实验中使用预规范化的 PopQA 评估，消歧影响较小，但现实场景可能显著。
- **覆盖范围受限于 KB**：若查询的实体或关系不在 KB 中，模型回退到参数记忆，此时事实性无法保证。KB 的完整性和质量成为瓶颈。
- **语言和领域限制**：仅测试英文和通用 / 生物医学 KB，未验证多语言或其他专业领域。
