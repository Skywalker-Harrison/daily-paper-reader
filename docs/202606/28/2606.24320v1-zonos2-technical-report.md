## 摘要
我们提出了ZONOS2 8B，这是我们最新的TTS模型，在自然度、韵律和声音克隆保真度方面达到了最先进水平。我们在规模、数据和训练方法上对Zonos-v0.1进行了改进。通过采用新颖的混合专家（MoE）架构，我们将模型从1.6B参数扩展到8B总参数（900M活跃参数），提升了推理延迟和吞吐量。我们使用新的数据处理流水线将训练语料从20万小时扩展到超过600万小时，并简化了后处理和条件设置方法以提高自然度和声音克隆保真度。我们在质量、说话人相似度、WER以及我们新颖的TTS基准测试ZTTS1-Eval上评估了ZONOS2 8B，它在保持良好流式延迟的同时与最先进的系统具有竞争力。我们在GitHub和Hugging Face上以Apache 2.0许可证发布模型权重和示例推理代码。

## Abstract
We present ZONOS2 8B, our latest TTS model, which achieves state-of-the-art naturalness, prosody, and voice cloning fidelity. We improve upon Zonos-v0.1 across scale, data, and training recipe. We scale the model from 1.6B to 8B total parameters (900M active) with a novel mixture-of-experts (MoE) backbone, improving inference latency and throughput. We expand our training corpus from 200K to over 6M hours using a new data processing pipeline, and we simplify our post-training and conditioning recipes to improve naturalness and voice cloning fidelity. We evaluate ZONOS2 8B on quality, speaker similarity, WER, and ZTTS1-Eval, our novel TTS benchmark, where it performs competitively with state-of-the-art systems while maintaining good streaming latency. We release our model weights and example inference code under an Apache 2.0 license on GitHub and Hugging Face.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：现有 TTS 系统往往在生成稳定性、自然度、可控性、多语言能力和推理性能之间难以兼顾：有些质量高但控制少，有些推理快但仅支持少数语言。
- **背景**：此前开放源 TTS 模型（如 Zonos-v0.1）在自然度和声音克隆保真度上仍有提升空间，且缺乏大规模、多语言的高质量训练数据与高效的推理架构。
- **目标**：设计一个全面强的 TTS 系统，**重点提升自然度和声音克隆保真度**，同时保持低流式延迟，并开源推动社区发展。

## 2. 论文提出的方法论

### 核心思想
- 采用 **Decoder-only Transformer + Mixture-of-Experts（MoE）** 架构，将模型总参数扩展到 8B，其中活跃参数仅 900M，兼顾能力与推理速度。
- 使用 **Residual Vector Quantization (RVQ) 音频分词器（DAC）** 进行高质量音频编码；采用 **字节级文本分词**（UTF-8 bytes），避免传统音素化（G2P）带来的多语言失败问题。
- 引入 **延迟模式（delay pattern）** 并行生成多个码本，降低流式解码延迟。
- 条件控制包括：**说话人嵌入（ECAPA-TDNN + LDA 投影）**、语速选择、质量模式（通过数据增强和条件 token 实现）。

### 关键技术细节
- **音频分词**：DAC 编码器 + 9 个码本的 RVQ，延迟模式：`Y[t, j] = X[t-j, j]`（j 延迟 j 帧）。
- **文本分词**：输入字符串直接编码为 UTF-8 字节序列，无需 G2P。
- **说话人嵌入**：通过 ECAPA-TDNN 提取 2048 维嵌入，经 LDA 投影至 1024 维，再映射到 transformer 隐藏维度。LDA 投影去除噪声、词汇、时长等非说话人信息，防止模型“作弊”。
- **语速条件**：文本字节数/时长，量化后作为 token 追加。
- **质量条件**：训练时对克隆音频施加背景噪声、压缩、混响等增强；引入“Augmented Embedding” token；在最终阶段增加“Quality Mode” token。
- **MoE Transformer**：28 层，隐藏维度 2048，前 3 层和最后 1 层为密集层，其余为 MoE 层（16 个专家，top-1 路由，最后 MoE 层 top-2）。路由设计借鉴 ZAYA1-8B（Exponential Depth Averaging + headwise gating）。注意力为 GQA（16 查询头，4 键值头）。
- **训练目标**：标准的自回归条件语言模型，预测延迟 DAC 令牌。损失为遮蔽负对数似然 + 路由平衡辅助损失。logits 使用 soft-cap（τ=15）。
- **优化器**：Muon（基学习率 5e-4，Muon 学习率 5e-3）。路由偏置使用独立的 AdamW。

## 6. 论文的主要结论与发现

- **ZONOS2 在自然度、韵律、声音克隆保真度上达到 SOTA**，在 ZTTS1-Eval Clean 和 ITW 上都表现竞争性能，尤其 ITW 韵律最佳（TTSDS2 最好）。
- **MoE 架构** 有效平衡模型能力与推理效率（8B 总参数，900M 活跃），首次成功应用于开源 TTS。
- **字节级分词** 优于音素级：消除 G2P 失败，提升多语言鲁棒性。
- **质量模式** 可显著提高 WER 和 UTMOS，但会略微降低说话人相似度；用户可按需选择。
- **说话人嵌入的 LDA 投影 + 两阶段训练** 有效缓解过拟合和因果泄露。
- 在 CosyVoice 3 Eval 和 Seed-TTS-Eval 上，ZONOS2 也取得有竞争力的成绩（如 Test-EN WER 2.05%）。
- 模型权重和部分代码开源。

## 7. 优点

- **开源并采用 Apache 2.0 许可**，促进社区复现与改进。
- **首次将 MoE 引入开放 TTS**，展示了大参数+小活跃的范式优势。
- **ZTTS1-Eval 基准** 弥补现有基准不足：涵盖更多语言、使用最新 ASR/质量模型、增加韵律和多样性度量。
- **数据处理流水线**：多 ASR 集成过滤 + 不同阶段阈值调整，有效提升训练数据质量。
- **说话人嵌入处理**：LDA 投影减少信息泄露，减少条件“作弊”，延长有用训练步数。
- **质量条件设计**：通过数据增强和 token 控制，使用户可权衡可懂度和相似度。

## 8. 不足与局限

- **MoE 训练稳定性问题**：在音频数据上路由平衡比文本更难，需手动调整第一/最后层为密集层、增加 top-2 路由等，未能完全解决不稳定现象（部分层熵降至 0.6）。
- **说话人嵌入仍存在过拟合倾向**：即使经过 LDA 和两阶段训练，作者仍指出“usable training horizon”有限。
- **质量模式存在取舍**：提高可懂度和质量的同时降低说话人相似度，未必适合所有场景。
- **语言覆盖侧重英语**：训练数据中英语占绝大多数，低资源语言表现可能仍有限（如泰国语、他加禄语 WER 较高）。
- **客观指标为主**：缺乏真人听感 MOS 测试，UTMOS 和 WER 不能完全反映自然度。
- **基准数据冲突风险**：无法确认闭源基线是否使用了 ZTTS1-Eval 中的数据进行训练。
- **计算资源未公开**：不利于复现和成本估算。
