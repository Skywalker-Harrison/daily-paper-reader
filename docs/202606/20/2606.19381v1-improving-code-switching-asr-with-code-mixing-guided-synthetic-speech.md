## 摘要
语码转换自动语音识别（ASR）因缺乏高质量语码转换文本-语音对用于训练而仍然具有挑战性。尽管已探索通过文本到语音（TTS）进行合成数据增强，但现有语码转换TTS方法主要优化重构保真度，并未明确强制语言边界一致性，从而限制了其对语码转换ASR增强的有效性。本文提出一种语码混合引导的偏好学习框架，利用语码混合指数（CMI）引导合成语音生成，提高语码转换保真度。在SEAME普通话-英语对话语料库上的实验表明，所提方法增强了合成数据用于ASR微调的实用性。具体而言，在微调Whisper Large时，该方法在DevMAN和DevSGE集上将混合错误率（MER）分别从12.1%/17.8%降至8.9%/14.2%。

## Abstract
Code-switch (CS) Automatic Speech Recognition (ASR) remains challenging due to limited availability of high quality CS text-speech pairs for training. Although synthetic data augmentation via Text-to-speech (TTS) has been explored, existing CS TTS approaches primarily optimise reconstruction fidelity and do not explicitly enforce language-boundary consistency, thereby limiting their effectiveness for CS ASR augmentation. This paper proposes a code-mixing guided preference-learning framework that steers synthetic speech generation toward improved code-switching fidelity using the Code Mixing Index (CMI). Experiments on the SEAME Mandarin-English conversational corpus demonstrate that the proposed method enhances the utility of synthetic data for ASR fine-tuning. Specifically, when fine-tuning Whisper Large, the proposed approach reduces Mixed Error Rate (MER) from 12.1%/17.8% to 8.9%/14.2% on the DevMAN and DevSGE sets, respectively.

## 2. 方法论：核心思想、关键技术细节
### 2.1 核心思想
- 将**文本级CMI**扩展到**语音级CMI_speech**，利用伪帧级语言标签量化合成语音中的语言混合程度。
- 基于CMI_speech计算ΔCMI（合成与真实语音的CMI差异），作为偏好信号之一，结合ASR识别准确率（MER）和感知质量（UTMOS）构建多critic评分，通过DPO（直接偏好优化）对齐TTS生成过程，使合成的CS语音更保真。

### 2.2 关键技术细节
#### （a）语音级CMI_speech
- 定义：CMI_speech(u) = (T(u) - max_{k∈L} T_k(u)) / T(u)，其中T(u)为总帧数，T_k(u)为语言k的帧数。
- 获取帧级语言标签：采用Whisper模型附加语言对齐损失（LAL），通过解码器交叉注意力将标记级语言身份映射到帧级。

#### （b）DPO偏好学习框架（三阶段）
- **阶段1**：基础微调TTS模型（CosyVoice2）至SEAME数据。
- **阶段2**：DPO对齐  
  - **候选生成**：对每个文本，通过随机采样生成N个候选语音。  
  - **Critic评分**：使用三个自动critic：  
    - MER（Whisper ASR识别错误率，越低越好）  
    - UTMOS（预测MOS分，越高越好）  
    - ΔCMI（合成与真实语音的CMI_speech差异，越小越好）  
  - **偏好选择**：将各critic归一化后加权得到综合得分R(Ŷ) = λ·S_UTMOS - γ·S_MER - ν·S_ΔCMI，选择最高分和最低分作为偏好对。对不可靠样本（MER>20%、UTMOS<2.5、ΔCMI>20%）进行过滤。
- **阶段3**：使用优化后的TTS生成合成语音，与原真实数据混合微调下游CS ASR模型。
