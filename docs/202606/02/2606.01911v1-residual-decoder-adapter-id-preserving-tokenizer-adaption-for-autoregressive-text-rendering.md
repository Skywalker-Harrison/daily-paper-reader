## 摘要
视觉自回归（AR）模型通过预测由视觉分词器解码的离散标记来生成图像。尽管展示了强大的整体图像生成能力，但在文本渲染上仍表现不佳，出现模糊笔画并破坏字母形状。本工作中，我们将这一局限性归因于视觉分词器难以重建细粒度细节。改进分词器虽然直接但成本高昂，因为需要重新训练分词器和AR模型。我们能否在不重新训练现有分词器和AR模型的情况下提升AR模型的文本渲染性能？为此，我们提出了残差解码器适配器（RDA），它在不改变分词器标记空间的情况下事后升级现有分词器。具体而言，它通过引入两个新颖组件来细化视觉分词器的解码器输出：（i）一个与原始分词器共享标记分布的配对码本；（ii）一个并行分支，用于学习像素空间中重建图像与真实图像之间的微小差异（残差）。这种残差设计使我们能够非侵入性地增强分词器，同时保持与先前AR模型的兼容性。RDA显著大幅提升了文本渲染性能。例如，在竞争性的TextAtlas基准上，我们将微调后的Janus-Pro OCR准确率从24.52%提升至58.26%（TextVisionBlend），从12.75%提升至36.81%（StyledTextSynth）。代码可在 https://github.com/CSU-JPG/RDA 获取。

## Abstract
Visual Autoregressive (AR) models generate images by predicting discrete tokens that are decoded by a visual tokenizer. Despite demonstrating strong overall image generation ability, they still underperform on text rendering with blur strokes and disrupt letter shapes. In this work, we trace this limitation to the visual tokenizer, which struggles to reconstruct fine-grained detail. Improving the tokenizer is straightforward but expensive, as it necessitates retraining both the tokenizer and the AR model. Can we improve text rendering performance of AR models without retraining the existing tokenizer and AR model? To achieve this, we propose the Residual Decoder Adapter(RDA) that upgrades an existing tokenizer post-hoc without changing its token space. Specifically, it refines the decoder output of the visual tokenizer by introducing two novel components: (i) a paired codebook that shares the token distribution with the original one; (ii) a parallel branch to learn the tiny differences (residual) between the reconstructed image and the ground-truth images in the pixel space. This residual design allows us to enhance the tokenizer non-invasively while preserving compatibility with prior AR models. RDA substantially improves text rendering significantly by a large margin. For instance, we boost finetuned Janus-Pro OCR accuracy rises from 24.52% to 58.26% (TextVisionBlend), from 12.75% to 36.81% (StyledTextSynth) on competitive TextAtlas benchmark. The code is available at https://github.com/CSU-JPG/RDA

# 论文详细总结

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：视觉自回归（AR）模型（如Janus-Pro、TAR）在图像生成上表现强，但文本渲染质量差——产生模糊笔画、字母形状扭曲。根源在于视觉分词器（tokenizer）离散量化过程丢失了细粒度高频细节，导致重建图像文本模糊。
- **背景**：现有解决方案需要重新训练分词器（如增大码本、层次量化等），但这会改变token ID空间，使已训练好的AR模型失效，必须重新训练整个AR模型，成本极高（数百GPU小时）。因此，急需一种**无需修改token ID空间、无需重训练AR模型**的提升文本渲染的方法。

## 2. 方法论：核心思想与关键技术细节
- **核心思想**：在**不改变原始分词器token ID分布**的前提下，通过附加一个轻量级插件模块RDA，学习原始重建图像与真实图像之间的像素级残差，从而精细恢复文本细节。
- **主要组件**：
  1. **共享ID提示码本（Shared-ID Hint Codebook）**：与原始码本（Z）大小相同、索引相同，但嵌入向量可学习。给定同一个token id，可从两个码本分别取出`f_base`（冻结，保留语义）和`f_hint`（可训练，提供高频细节）。保证token分布不变，AR模型可直接复用。
  2. **残差解码器（Residual Decoder）**：以混合特征`f_mix = p(f_base) + q(f_hint)`为输入，预测像素级残差`ΔÎ`，最终重建图像为`Î_final = Î + ΔÎ`。残差解码器结构与原解码器类似，最后两层卷积通道加倍以捕捉高频信息。
  3. **训练目标**：冻结原始分词器，仅优化RDA参数。损失函数包括：
     - 重建损失（MAE+MSE）：`L_rec`
     - 最终图像感知损失：`L_final_perc`
     - **残差感知损失**（关键）：`L_res_perc = ||ϕ(ΔI) - ϕ(ΔÎ)||²`，促使模型学习结构化残差
     - 边缘感知Sobel损失：`L_sobel = |M_edge ⊙ (ΔI - ΔÎ)|₁`
     - 频率域损失：`L_freq`，保留高频成分
  - 总损失：`L_total = L_rec + L_final_perc + L_res_perc + L_sobel + L_freq`。
- **推理流程**：AR模型生成离散token序列→从原始码本和提示码本同时查找特征→投影融合→残差解码器预测残差→加到原始解码图像上得到最终结果。

## 3. 实验设计

  - **评估**：五个文本生成基准：Mario-Eval、AnyText-Benchmark、LongTextBench、CVTG-2K、TextAtlasEval（含StyledTextVisionBlend、StyledTextSynth、TextScenesHQ）。
- **对比方法**：
  - 基线AR模型：Janus-Pro（1B/7B）、TAR（1.5B/7B）、Lumina-mGPT（7B）。
  - 其他分词器：LlamaGen-VQ、Chameleon-VQ、TA-Tok、UniTok（使用1.28B训练数据）。
- **评估指标**：
  - 文本渲染：OCR准确率（Acc.）、F1、字符错误率（CER）。
  - 图像质量：SSIM、LPIPS、PSNR、FID。
- **实验场景**：
  - ① **生成任务**：将RDA应用于通用AR模型和文本专用AR模型（微调过TextAtlas），比较加/不加RDA的性能。
  - ② **重建任务**：比较LlamaGen-VQ、Chameleon-VQ等分词器加/不加RDA的重建指标。
  - ③ **消融实验**：损失函数组合、码本设计（维度、有无码本）、特征注入方式（Add/Cross-Attn/Concat+Conv）、与直接微调解码器对比。

## 4. 资源与算力
- **训练配置**：64张V100 GPU（8节点×8GPU），全局batch size=512，训练120k步，混合精度，AdamW优化器（β1=0.9, β2=0.95），学习率1e-4，梯度裁剪1.0。

## 6. 主要结论与发现
- RDA能**显著提升AR模型文本渲染质量**：在通用AR模型（Janus-Pro 7B）上，AnyText-Benchmark OCR准确率从8.85%升至10.07%；在文本专用AR模型（微调后Janus-Pro 1B）上，StyledTextVisionBlend准确率从24.52%跃升至58.26%（+33.74点）。
- RDA在**重建任务**上也大幅提升OCR准确率（约10-15点），同时保持或改善SSIM/LPIPS。
- 相比直接微调解码器，RDA拥有更好的**OOD泛化性**：在ImageNet验证集上FID仅上升1.57（9.63→11.20），而直接微调解码器导致FID恶化5.12（9.63→14.75）。
- 残差感知损失是最关键组件（去除后训练失败）；共享ID提示码本在维度16时最佳；特征注入不可缺失。

## 7. 优点
- **非侵入式、即插即用**：无需修改原tokenizer和AR模型权重，直接作为插件提升文本渲染，兼容所有基于VQ-VAE的AR模型。
- **高效**：仅用5M训练数据、64V100 GPU 120k步，即可取得显著效果，训练成本远低于重新训练分词器+AR模型。
- **跨分辨率泛化**：仅在256分辨率训练，可直接推广到384/512/1024，无需微调。
- **代码开源**：促进后续研究与复现。

## 8. 不足与局限
- **未解决AR模型自身预测错误**：当AR模型生成错误的token序列（如缺少字符、结构错误）时，RDA无法纠正，仅能改善解码视觉质量。
- **对极小或模糊文本效果有限**：当字符尺寸过小或视觉歧义时，tokenizer本身分配错误token，残差无法恢复。
- **训练数据单一**：仅使用Mario-10M子集，未验证在其他分布（如文档图像、手写体）下的泛化能力。
- **实验覆盖缺失**：未在更多样化的tokenizer（如RQ-VAE）上测试，也未与最新DiT头方法进行公平对比。
- **计算开销**：虽然端到端延迟增加小，但RDA模块本身参数较多（~165M），对资源受限场景可能构成负担。
- **长期兼容性**：若未来AR模型采用不同的tokenization策略（如软量化、连续表示），RDA的设计需要适配。
