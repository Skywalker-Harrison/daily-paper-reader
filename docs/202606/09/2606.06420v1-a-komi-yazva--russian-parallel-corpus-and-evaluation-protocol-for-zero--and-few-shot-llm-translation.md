## 摘要
我们首次提出了科米-亚兹瓦语-俄语平行语料库，以及一个明确的评估协议，用于在濒危、极低资源环境下研究大语言模型的翻译。该数据集包含来自74篇叙事文本的457个对齐句对，并附有文档化的来源信息、句子级对齐和故事标识符，从而支持泄漏感知评估。我们利用这一设置，在零样本和基于检索的少样本场景下，比较了现代大语言模型在严重平行数据稀缺情况下的科米-亚兹瓦语到俄语翻译。该协议包括故事级交叉验证、用于少样本提示的确定性检索、生成输出的严格验证、互补的基于参考和基于评判的指标，以及故事级不确定性估计。在各模型中，大语言模型产生了非平凡的翻译，但性能因模型家族和提示策略而异。基于检索的少样本提示始终优于零样本提示，而在小检索上下文之外的增益仍然有限。结果表明，在这种设置下，评估结论很大程度上取决于指标选择和失败处理方式，因此本文将该语料库既视为数据集贡献，也视为濒危语言机器翻译的可重复评估测试平台。

## Abstract
We present the first Komi-Yazva--Russian parallel corpus together with an explicit evaluation protocol for studying LLM translation in an endangered, extremely low-resource setting. The dataset contains 457 aligned sentence pairs from 74 narrative texts and is accompanied by documented provenance, sentence-level alignment, and story identifiers that enable leakage-aware evaluation. We use this setup to compare modern large language models on Komi-Yazva-to-Russian translation under severe parallel-data scarcity in zero-shot and retrieval-based few-shot regimes. The protocol includes story-level cross-validation, deterministic retrieval for few-shot prompting, strict validation of generated outputs, complementary reference-based and judge-based metrics, and story-level uncertainty estimates. Across models, LLMs produce non-trivial translations, but performance varies strongly by model family and prompting regime. Retrieval-based few-shot prompting consistently improves over zero-shot prompting, while gains beyond a small retrieved context remain limited. The results show that evaluative conclusions in this setting depend materially on metric choice and failure handling, so the paper frames the corpus as both a dataset contribution and a reproducible evaluation testbed for endangered-language machine translation.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建一个极小规模但结构良好的平行语料库，并设计包含故事级防泄漏、确定性检索、严格输出验证、多指标互补和故事级不确定性估计的评估协议，用于公平比较LLM在零样本和少样本提示下的翻译表现。
- **关键技术细节**：
  - **语料库构建**：从语言学专著《科米-亚兹瓦方言》（Lytkin, 1961）中数字化并手动对齐得到457个句对，来自74篇叙事文本。使用`id_story`和`id_paragraph`两级标识符，支持故事级分组。
  - **评估协议**：
    - **故事级交叉验证**：5折GroupKFold，按`id_story`分组，确保同一故事的所有句子完全属于同一折，避免训练/测试之间的内容泄漏。
    - **确定性检索**：在训练集源端使用TF-IDF（字符级n-gram，范围3–5）为每个测试句子检索最相似的k个源句，对应的参考译文作为少样本示例。避免随机采样，确保可复现。
    - **提示格式**：统一的`SOURCE: ... RUSSIAN: ...`对格式，加上系统指令：“你是专业翻译，只输出俄语翻译，无解释、无引号、无多余行。” 对部分模型关闭隐藏推理。
    - **生成验证与失败处理**：检测提示泄漏、源句复制、非俄语输出、格式化错误等，区分传输失败和内容级失败。传输失败重试，最终仍失败的计入错误。
    - **评估指标**：主要指标为chrF（字符级，适合形态丰富语言），辅以BLEU、TER和LLM-as-a-judge分数（使用GPT-4.1-mini）。

## 6. 论文的主要结论与发现

1. **模型选择至关重要**：模型家族之间的性能差异巨大，最强（Gemini 3.1 Pro）与最弱（Qwen3 30B A3B）的chrF相差约30个百分点。通用性能好的模型并不自动保证在该低资源方向上表现优异。
2. **少样本提示显著优于零样本**：从k=0到k=4，所有指标大幅提升（LLM-as-a-judge平均分从1.54升至2.23，chrF从19.72升至31.63），且该增益在所有模型和所有折叠中一致。
3. **增加演示数量收益递减**：从k=4到k=8，增益很小（chrF仅从31.63升至32.96，Judge分数从2.23升至2.34）。TER在k=8时反而变差（从85.11升至96.09），提示更大提示可能导致更多局部编辑需求。
4. **评估指标与失败处理影响结论**：不同指标（chrF、BLEU、TER、Judge、成功行数）给出的模型排序不完全一致。例如Gemini 3.1 Pro在质量指标上领先，但成功率低于Claude Sonnet 4.6。评估结论必须明确指出所依赖的指标和失败处理方式。
5. **质量-可靠性权衡**：最佳模型（Gemini 3.1 Pro）和最佳平衡模型（Claude Sonnet 4.6）之间存在取舍，实用部署需根据具体场景选择。
