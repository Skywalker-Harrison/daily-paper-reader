## 摘要
当前对视觉语言模型（VLM）的多语言评估假设语言与正字法之间存在一一对应的映射关系，忽略了数十亿使用多文字语言的用户。我们引入了PuMVR（旁遮普语多模态视觉推理），这是一个包含旁遮普语三种活跃文字（古尔穆基文、沙穆基文和罗马文）1000个严格平行的图像-文本实例的基准测试。评估了10个最先进的VLM，我们发现了一个显著且系统性的文字差距。模型经常能在一个文字中解决视觉任务，但在另一个文字中却无法完成相同的任务，准确率差距高达16%。关键的是，视觉输入能够统一地提升绝对性能，但并不能缩小正字法差距。此外，跨文字的上下文内迁移非常脆弱，暴露了文字锁定的知识表示。通过所有文字对的McNemar检验支持，我们的发现表明当前的“多语言”VLM并非真正的多文字。我们提出了文字一致性率（SCR），在我们的基准测试中低至24.8%，作为文字无关评估的强制性指标，以确保公平的AI访问。数据和代码可在：https://github.com/prabhjotschugh/Not-Truly-Multilingual-PuMVR。

## Abstract
Current multilingual evaluations for Vision-Language Models (VLMs) assume a one-to-one mapping between language and orthography, overlooking billions of users of multi-script languages. We introduce PuMVR (Punjabi Multimodal Visual Reasoning), a benchmark of 1,000 strictly parallel image-text instances across Punjabi's three active scripts: Gurmukhi, Shahmukhi, and Roman. Evaluating 10 state-of-the-art VLMs, we expose a substantial and systematic Script Gap. Models frequently solve visual tasks in one script while failing identical tasks in another, with accuracy deltas reaching 16%. Crucially, visual input boosts absolute performance uniformly yet does not close the orthographic gap. Furthermore, cross-script in-context transfer is highly brittle, exposing script-locked knowledge representation. Supported by McNemar tests across all script pairs, our findings demonstrate that current "multilingual" VLMs are not truly multi-script. We propose the Script Consistency Rate (SCR), which falls as low as 24.8% on our benchmark, as a mandatory metric for script-agnostic evaluation to ensure equitable AI access. Data and code are available at: https://github.com/prabhjotschugh/Not-Truly-Multilingual-PuMVR.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前多语言视觉语言模型（VLM）的评估建立在“一种语言、一种文字”（OLOS）的隐性假设上，即语言与书写系统一一对应。这忽略了全球数十亿使用**多脚本语言**（如旁遮普语同时使用古尔穆基文、沙穆基文和罗马文）的用户，导致评估结果无法反映模型在真实多脚本场景下的实际能力。
- **研究动机**：现有基准（如XM3600、MaXM等）仅用单一脚本表示每种语言，无法捕捉模型在不同脚本下处理相同语义内容时的性能差异。这种“脚本盲区”会造成对模型多语言能力的虚假高估，并带来严重的公平性问题。
- **整体含义**：论文指出，当前“多语言”VLM并非真正的“多脚本”系统，模型的知识表示可能高度依赖正字法模式而非语义概念。因此，必须将**脚本一致性**作为评估多语言能力的一个独立维度，才能确保AI对多脚本社区用户的公平访问。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：通过构建**严格平行的多脚本图文基准**，将脚本作为唯一的独立变量，隔离其他因素（图像、语义内容、任务难度）的影响，从而量化模型在不同脚本下的性能差异。
- **关键技术细节**：
  - **PuMVR基准**：包含1000个平行实例，每个实例由一张图像、一个问题和四个选项组成。问题和选项分别用古尔穆基文（Gurmukhi）、沙穆基文（Shahmukhi）和罗马文（Roman）三种脚本呈现，且语义完全等价。所有实例经两位独立母语专家验证，语义等价性、答案正确性和脚本准确性的PABAK系数均≥0.97。
  - **三项实验**：
    1. **零样本脚本差距量化**：分别在三种脚本下评估模型，计算每脚本准确率、脚本一致性率（SCR）和性能差Δ（最大跨脚本准确率差）。SCR定义为同时正确回答所有三个脚本的实例比例。
    2. **模态重要性消融**：比较纯文本（无图像）和多模态（有图像）两种输入条件下的准确率，计算视觉增益（VG），检验视觉信息能否弥补脚本差异。
    3. **跨脚本少样本迁移**：使用k=3个上下文示例，测试同脚本、跨脚本和混合脚本三种条件，计算少样本提升（FSL）和迁移效率（TE），评估模型知识是否被脚本锁定。
- **公式**（文字说明）：
  - 脚本准确率：Acc_s = 正确预测数 / 总实例数
  - SCR = 三个脚本均正确的实例数 / 总实例数
  - Δ = max(|Acc_s1 - Acc_s2|)
  - VG_s = Acc(多模态, s) - Acc(纯文本, s)
  - TE_{T→S} = Acc(少样本, T→S) / Acc(少样本, S→S) × 100%
- **统计验证**：对每对脚本比较使用McNemar检验，确认性能差距的统计显著性（8/10模型在古尔穆基文-沙穆基文对上显著，其中6个p<0.001）。

## 3. 实验设计

- **数据集与场景**：使用**PuMVR基准**的375个实例（分层抽样固定为评估集，剩余625个保留用于未来微调）。场景涵盖视觉推理任务（如类比、常识推理等），图像为文化相关或通用内容。
- **Benchmark**：PuMVR本身是新构建的基准，作为评估平台。
- **对比方法**：评估了10个最先进的VLM，包括：
  - 专有前沿模型：GPT-4o（OpenAI）、Gemini 2.5 Flash（Google DeepMind）、Claude Sonnet 4（Anthropic）、Grok-4-1-fast-reasoning（xAI）。

- **对比方式**：所有模型在同一375实例集上按相同实验协议进行评估，确保公平比较。

## 6. 主要结论与发现

1. **脚本差距普遍且显著**：所有10个模型均表现出脚本依赖的性能下降。跨脚本准确率差Δ最高达16.26%（Llama-3.2-11B-Vision），最低也有4.11%（Qwen2-VL-72B-Instruct）。多数模型在古尔穆基文上最优，沙穆基文最差。
2. **脚本一致性率（SCR）极低**：最佳模型Gemini 2.5 Flash的SCR为85.60%，而最差Kimi-VL-A3B-Instruct仅24.80%。这意味着即使单脚本准确率高，模型也无法稳定地跨脚本正确回答。
3. **视觉输入提供平行增益，但不能弥补脚本差距**：模态消融显示视觉增益（VG）统一为正（7.5%~36.1%），但脚本之间的性能差距依然存在，视觉信息不能“修复”正字法的缺陷。
4. **跨脚本少样本迁移脆弱**：许多模型出现负的少样本提升（如Qwen2-VL-72B-Instruct在古尔穆基文同脚本条件下FSL达-49.9%），且迁移效率不对称（如沙穆基文→古尔穆基文的TE>150%，但反向仅81%），说明知识被脚本锁定，并非语义层面的共享表示。

## 7. 优点

- **问题重要性**：首次系统性地揭示多脚本VLM评估中的盲区，具有高社会影响（影响数十亿用户）。
- **实验设计严谨**：通过平行脚本严格控制语义和视觉变量，McNemar检验提供统计保障，避免了常见基准中混淆语言与脚本的问题。
- **评估全面**：覆盖主流专有和开放权重模型，包含零样本、消融和少样本三种典型场景，全面刻画了脚本差距的性状。
- **提出新指标SCR和TE**：SCR作为跨脚本一致性的简洁度量，TE揭示了迁移方向性不对称，具有推广至其他多脚本语言的价值。
- **数据质量和透明度高**：人工验证的平行语料质量高，代码、数据、提示模板全部开源，便于复现和扩展。

## 8. 不足与局限

- **语言覆盖有限**：仅针对旁遮普语，其他多脚本语言（如塞尔维亚语、库尔德语、印地-乌尔都语）的模式可能不同，结论的泛化性需更多验证。
- **数据集规模较小**：评估仅使用375个实例，限制了细粒度统计分析和子类别（如不同推理类型）的探讨。
- **未涵盖微调场景**：实验仅限于零样本和少样本推理，未测试全微调或指令微调后脚本差距是否缩小。
- **潜在的混淆因素**：尽管尽力保证语义等价，但文化关联（如古尔穆基文与锡克教，沙穆基文与伊斯兰教）可能引入难以完全分离的偏差。
- **开放性模型覆盖不全**：未包含最近更新的模型（如Gemini 2.0系列、Claude 4等更新版本），且专有模型API版本可能随时间变化，影响复现性。
- **未提供多脚本文本检测或标注意见**：未评估模型在混合脚本输入下的表现（实际使用中常见脚本混用）。
