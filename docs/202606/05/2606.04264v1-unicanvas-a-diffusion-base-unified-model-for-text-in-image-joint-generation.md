## 摘要
近年来，在单一架构内同时处理多模态理解和生成的统一视觉语言模型取得了显著进展。自回归视觉语言模型能够跨模态推理，但无法生成高质量图像。相比之下，扩散模型能生成逼真的视觉内容，却难以生成连贯的文本，这使得开发一个既能无缝处理视觉生成又能处理文本生成的统一模型颇具挑战。最新研究表明，语言可以有效地嵌入到视觉表示中，使模型能够直接从图像中推理文本语义。为此，我们提出了UniCanvas，这是首次尝试通过文本嵌入图像的生成方式，统一扩散模型以生成交错的多模态内容。扩散模型天然地捕捉共享像素画布上的变换，这可以视为视觉变化的世界模型。该模型不生成离散的文本标记，而是学习将语言表示为图像中的视觉模式，利用其固有的多模态嵌入空间。这种设计使得模型在图像合成过程中能够在单个像素画布内自然“绘制”文本，实现无缝的多模态生成。实验表明，UniCanvas相比之前的统一模型性能有所提升，将文本嵌入图像的扩散模型生成确立为一种有前景的统一多模态生成范式。

## Abstract
Recent years have seen remarkable progress in unified vision-language models handling both multimodal understanding and generation within a single architecture. While autoregressive VLMs can reason across modalities, they fail to generate high-quality images. In contrast, diffusion models produce photorealistic visuals yet struggle to generate coherent text, making it challenging to develop a single unified model that can seamlessly handle both visual and text generation. Recent advances suggest that language can be effectively embedded within visual representations, allowing models to reason about textual semantics directly from images. To this end, we propose UniCanvas, a first attempt that unifies diffusion models to generate interleaved multimodal contents through text-in-image generation. Diffusion models naturally capture transformations on a shared pixel canvas, which can be viewed as world models of visual change. Instead of producing discrete text tokens, the model learns to represent language as visual patterns inside images, leveraging its inherent multimodal embedding space. This design allows the model to "draw" text naturally within a single pixel canvas during image synthesis, achieving seamless multimodal generation. Experiments demonstrate that UniCanvas improves performance over previous unified models, positioning text-in-image generation with diffusion models as a promising unified multimodal generation paradigm.

- **动机**：最新研究显示，语言可以有效地嵌入到视觉表示中（例如将文本直接渲染到图像上），使得模型能够直接从图像中推理文本语义。作者希望利用扩散模型在共享像素画布上自然“绘制”文本，实现文本与图像的联合生成，从而构建真正的统一多模态模型。
- **整体含义**：UniCanvas 首次尝试在纯扩散模型框架内实现文本嵌入图像（text-in-image）的生成，将语言视为视觉模式，消除模态鸿沟，为统一多模态生成提供新范式。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：**共享像素画布**——语言和视觉共享同一个像素空间，文本被渲染为图像中的视觉模式（光学文字），而不是离散 token。模型通过学习在画布上“写入”文本和更新场景状态，实现多步推理与视觉合成的统一。
- **关键技术细节**：
  - **Flow-Matching 框架**：将文本生成和状态更新建模为从当前画布到目标画布的流匹配（velocity prediction）。定义源 latent \(x_0\) 和目标 latent \(x_1\)，训练速度网络 \(v_\theta(x_t, t, c)\) 预测 \(v_t = x_1 - x_0\)。
  - **两阶段生成**：
    - **第一阶段：视觉推理**（Stage 1）——在画布上生成语义文本（例如动作指令）。输入当前图像 \(x\)，以推理 prompt \(c_{\text{reason}}\) 为条件，通过流匹配得到带文本的画布 \(\hat{x}^{\text{text}}_0\)。
    - **第二阶段：视觉合成**（Stage 2）——基于带文本的画布生成下一状态图像。以执行 prompt \(c_{\text{exec}}\) 为条件，从 \(\hat{x}^{\text{text}}_0\) 出发流匹配到目标状态 \(x^{\text{state}}_0\)。
    - **多步迭代**：将 \(\hat{x}^{\text{state}}_0\) 作为下一轮的输入，交替执行两阶段，生成连贯的图像-文本序列。
  - **CLIP 对齐损失**：用预训练 CLIP（冻结）对中间画布 \(\hat{x}^{\text{text}}_0\) 与推理 prompt \(c_{\text{reason}}\) 进行对比学习，增强文本语义与渲染文本的一致性。总损失：\(\mathcal{L}_{\text{total}} = \mathcal{L}^{\text{text}}_{\text{FM}} + \mathcal{L}^{\text{state}}_{\text{FM}} + \lambda_{\text{CLIP}} \mathcal{L}_{\text{CLIP}}\)。
  - **预训练阶段**：在 VQA 数据上学习文本渲染（将答案作为光学文字画在图像底部），同时混合通用图像编辑数据以保持编辑能力。预训练中额外使用 LPIPS 感知损失对文字区域进行正则化。
- **推理流程**：第一阶段 40 步 Euler ODE 生成文本画布；第二阶段 40 步 Euler ODE 生成下一状态图像。每个步骤仅依赖当前画布，无需完整历史。

## 6. 主要结论与发现
- UniCanvas 在 VSP 多步规划任务上平均成功率 0.77，显著优于所有统一模型基线（最高 0.24）及 Qwen2.5-VL 微调（0.72）。

- 通用推理任务上（COCO-QA / Visual7W）表现与 Qwen2.5-VL-3B 有差距但仍具竞争力（0.58/0.78 vs 0.72/0.86），说明文本嵌入图像范式具备泛化能力。
- 证明**所有推理在统一的像素画布上进行**，避免了跨模态转换的误差累积，是实现长程稳定多步生成的关键。

## 7. 优点
- **方法创新**：首次在扩散模型内实现文本嵌入图像生成，将语言视为视觉模式，消除模态鸿沟；两阶段设计分离推理与合成，共享同一 backbone。
- **效果显著**：在需要多步视觉-文本推理的 VSP 任务上大幅超越所有统一模型，验证了范式有效性。
- **能力强**：具备自纠错能力，能在错误后恢复，显示出一定的鲁棒性和上下文理解。
- **设计简洁**：使用 Flow Matching 统一文本和图像生成，CLIP 对齐损失轻量且有效，预训练阶段巧妙利用 VQA 数据。

## 8. 不足与局限
- **长程误差累积**：在 6×6 网格等复杂场景下，多步滚动后状态预测可能出现偏差，视觉错误随时间放大。
- **文本生成质量**：在需要长句或长文本输出（如问答）时，文本渲染易出现幻觉、字符错误或重复，不如自回归 VLM 可靠。
- **推理速度慢**：扩散模型需要进行多步 ODE 积分，推理时间是自回归 VLM 的数倍。
- **通用推理弱**：在 COCO-QA / Visual7W 上仍落后于专用 VLM，说明当前训练数据和监督规模有限，尚不能处理复杂开放形式推理。
- **实验覆盖**：通用推理仅取 200 样本，统计显著性偏低；未在更多标准多模态基准（如 MMLU、VQA v2）上系统评估。
- **应用限制**：依赖于特定的图像编辑 backbone（Qwen-Image-Edit），模型可迁移性需进一步验证；固定画布大小和文字区域限制了复杂场景文字排版。
