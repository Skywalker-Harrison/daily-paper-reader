## 摘要
对于低资源非拉丁文字的语言，文本转语音（TTS）评估如果仅依赖单一ASR往返词错误率（WER）可能会失败。系统可能不产生音频、说出邻近语言、仅在ASR转录中保留目标脚本文本，或对母语听众听起来不自然。我们提出了INSV（可理解性、自然度、脚本保真度和验证）报告框架，以区分这些情况。本文报告了INSV-A，即自动化筛查子集：合成完成度、ASR WER/CER、转录脚本保真度以及音频语言识别。本版本指定了母语MOS和语音标注，但不声称已实现。我们将INSV-A实例化为PashtoTTS-Bench，这是一个针对普什图语TTS的时序基准。2026年4月至5月的运行评估了Edge GulNawaz、Edge Latifa、OmniVoice克隆、OmniVoice自动以及一个乌尔都语负对照，使用了200条FLEURS和200条过滤后的Common Voice 24提示。在独立的omniASR_CTC_300M_v2下，OmniVoice自动的WER最低（FLEURS为24.1%，CV24为27.4%），其次是Edge GulNawaz（32.8%，39.5%）、Edge Latifa（35.6%，47.7%）和OmniVoice克隆（45.4%，34.8%）。WER低于自然语音基线反映了干净的合成音频，不应被视为优于母语语音。Whisper Large V3在检查的普什图语TTS音频上返回0.0%的普什图语标签，而MMS-LID-4017和SpeechBrain VoxLingua107将普什图语输出与乌尔都语对照区分开来。该版本提供了提供商元数据、每句分数、LID审计、失败日志以及用于添加系统的脚本。

## Abstract
Text-to-speech (TTS) evaluation for low-resource non-Latin-script languages can fail when it relies on a single ASR round-trip word error rate (WER). A system may produce no audio, speak a neighbouring language, preserve target script text only in an ASR transcript, or sound unnatural to native listeners. We introduce INSV (Intelligibility, Naturalness, Script fidelity, and Verification), a reporting framework that separates these cases. This paper reports INSV-A, the automated screening subset: synthesis completion, ASR WER/CER, transcript Script Fidelity Rate, and audio language identification. Native MOS and phonetic annotation are specified but not claimed in this release. We instantiate INSV-A as PashtoTTS-Bench, a dated benchmark for Pashto TTS. The April-May 2026 run evaluates Edge GulNawaz, Edge Latifa, OmniVoice clone, OmniVoice auto, and an Urdu negative control on 200 FLEURS and 200 filtered Common Voice 24 prompts. Under the independent omniASR_CTC_300M_v2, OmniVoice auto has the lowest WER (24.1% FLEURS, 27.4% CV24), followed by Edge GulNawaz (32.8%, 39.5%), Edge Latifa (35.6%, 47.7%), and OmniVoice clone (45.4%, 34.8%). WER below the natural-speech baseline reflects clean synthetic audio and should not be read as better than native speech. Whisper Large V3 returns 0.0% Pashto labels on checked Pashto TTS audio, while MMS-LID-4017 and SpeechBrain VoxLingua107 separate Pashto outputs from the Urdu control. The release provides provider metadata, per-sentence scores, LID audits, failure logs, and scripts for adding systems.

- **整体含义**：提供一种可复现、可扩展的自动化筛查协议（INSV-A），用于低资源非拉丁文字 TTS 系统，并通过普什图语基准（PashtoTTS-Bench）实例化，为后续研究提供基线工具和失败分类法。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 INSV 报告框架
- 核心思想：将 TTS 评估分解为四个独立维度，每个回答一个关键问题：
  - **I（可理解性，Intelligibility）**：输出能否被 ASR 转录为目标文本 → 使用 ASR 往返 WER/CER。
  - **N（自然度，Naturalness）**：输出是否对母语者听起来自然 → 需要人工 MOS 分数（本文未报告）。
  - **S（脚本保真度，Script fidelity）**：ASR 转录是否保持在目标脚本内 → 使用脚本保真率（SFR）。
  - **V（语言验证，Verification）**：输出是否实际为目标语言 → 使用多个语言识别（LID）模型。

### 2.2 INSV-A：自动化筛查子集
- 本文报告的是 INSV-A，只包含自动化可测的维度（S、V、I 和合成完成度），不包含 N。
- **合成完成检查**：确认系统是否为每条提示生成了非零长度的音频文件。
- **ASR 往返评估**：
  - 使用两个 ASR 模型：**独立模型 omniASR_CTC_300M_v2**（主指标，避免作者偏差）和**语言特定模型 pashto-asr-v3**（由作者训练，作为辅助诊断）。
  - 计算 WER（词错误率）、CER（字符错误率）以及 Perfect%（零 WER 比例）和 Low-error%（WER≤10% 比例）。
  - 使用 Bootstrap 95% 置信区间（1000 次重采样）。
- **脚本保真度（SFR）**：
  - 定义为 ASR 假设中属于目标脚本的字符数除以可计数字符数。
  - 对于普什图语，目标脚本为 Unicode U+0600–U+06FF 及呈现形式。
  - SFR=1.0 表示完全在正确脚本内；SFR≈0 表示脚本替换。
- **语言验证**：
  - 使用三个自动 LID 模型：Whisper Large V3、MMS-LID-4017、SpeechBrain VoxLingua107。
  - Whisper 被排除（因为对普什图语返回 0.0% 标签），仅保留作诊断。
  - 若模型间冲突，标记为“未解决”，需要母语标注者裁决。

### 2.3 失败分类法（F1–F5）
- **F1：预合成拒绝** —— 系统不产生音频（例如 API 错误）。
- **F2：语言替换** —— 系统输出非目标语言（如用乌尔都语替代普什图语）。
- **F3：音位崩塌** —— 系统 neutralize 了普什图语特有音位（如侧擦音变为普通擦音）。
- **F4：韵律不流畅** —— 语音正确但听感不自然（平调、节奏异常）。
- **F5：字形歧义** —— 系统混淆了视觉相似的字母（如 ي 与 ې）。

### 2.4 算法流程（文字描述）
1. 给定提示文本 T，调用 TTS 系统 S 生成音频 A = S(T)。
2. 检查 A 是否非空（否则记为 F1 失败）。
3. 使用 ASR 模型 R 转录 A 得到假设 H = R(A)。
4. 计算 WER = edit_distance(T_norm, H_norm)/|T_norm|（经 NFC 标准化、移除变音符号等）。
5. 计算 SFR = 属于目标脚本的字符比例。
6. 运行 LID 模型，获取语言标签，检查是否为目标语言。
7. 对各维度报告分数，并根据阈值和模型一致性判断通过/未通过/未解决。

- **语言识别结果**：

- **失败分类**：
  - 乌尔都语为确认的 F2（语言替换）。
  - GulNawaz 在三个 FLEURS 句子中出现 U+06CC→U+06D0 系统替换，候选 F5（字形歧义），但可能是参考文本标准化问题而非 TTS 发音错误。
  - 其他 F3、F4 均未确认，需母语标注。

## 7. 优点

1. **框架可推广性**：INSV 框架不限于普什图语，可迁移到其他非拉丁文字语言，只需提供特定于语言的输入（提示集、ASR 模型、LID 模型、脚本范围、字素列表）。
2. **多维评估**：分离可理解性、自然度、脚本保真度和语言验证，避免单一 WER 指标的盲区。
3. **失败分类法**：提出 F1–F5 分类法，系统化描述低资源非拉丁文字 TTS 的典型失败模式，具有实用指导价值。
4. **可复现性**：公开固定提示、脚本、元数据、哈希值；商业音频虽不发布的但可重新合成，保证未来对比。
5. **独立性考量**：使用独立 ASR 模型 omniASR 作为主指标，缓解了作者训练 ASR 和数据集导致的偏差风险。
6. **对工具局限性的诚实报告**：清晰指出 Whisper 无法识别普什图语、MMS 与 VoxLingua107 存在分歧，建议多模型融合并保留人工裁决。

## 8. 不足与局限

1. **系统覆盖有限**：仅评估了 2 个商业系统 + 2 个开源系统的两种模式；缺失 Google、OpenAI、ElevenLabs 等常见多语言服务，因为这些服务未验证普什图语支持。
2. **未进行人工评估**：MOS（自然度）和母语音位标注均未完成，因此对 F3、F4、F5 的结论仅为候选，缺乏确认。论文自身也强调“INSV-A 不代替人工听证”。
3. **潜在偏差**：作者训练了 pashto-asr-v3 模型和 Edge TTS 数据集，尽管使用独立 ASR 作为主指标，但辅助诊断的后端为作者模型，可能影响细致分析（如字素类 WER 分析）。
4. **实验数量偏少**：每条提示只生成一次音频（无多次重复），未讨论 TTS 随机性（如采样温度等）。
5. **SFR 未发挥区分力**：因本次评估的所有系统均输出正确脚本，SFR 成为常数，论文承认“只有在无普什图语路径的系统上才会有区分信号”。
