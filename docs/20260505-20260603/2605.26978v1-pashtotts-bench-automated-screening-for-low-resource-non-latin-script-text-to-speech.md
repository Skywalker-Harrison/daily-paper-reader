## 摘要
当仅依赖单一ASR往返词错误率（WER）时，针对低资源非拉丁文字语言的文本转语音（TTS）评估可能失败。系统可能不输出音频、说出邻接语言、仅在ASR转录中保留目标文字文本，或对母语听者而言听起来不自然。我们提出了INSV（可懂度、自然度、文字忠实度与验证）报告框架，将这些情况区分开来。本文报告了INSV-A自动化筛选子集：合成完成度、ASR WER/CER、转录文字忠实度比率以及音频语言识别。母语MOS和语音标注已指定但未在本版本中声明。我们将INSV-A实例化为PashtoTTS-Bench，一个带有时间标注的普什图语TTS基准。2026年4月至5月的运行评估了Edge GulNawaz、Edge Latifa、OmniVoice克隆版、OmniVoice自动版以及一个乌尔都语阴性对照，使用200个FLEURS提示和200个经过筛选的Common Voice 24提示。在独立的omniASR_CTC_300M_v2上，OmniVoice自动版取得了最低的WER（FLEURS 24.1%，CV24 27.4%），其次是Edge GulNawaz（32.8%，39.5%）、Edge Latifa（35.6%，47.7%）和OmniVoice克隆版（45.4%，34.8%）。WER低于自然语音基线反映了清晰的合成音频，不应被解读为优于自然语音。Whisper Large V3在已检查的普什图语TTS音频上返回0.0%的普什图语标签，而MMS-LID-4017和SpeechBrain VoxLingua107将普什图语输出与乌尔都语对照区分开来。该发布提供了提供者元数据、每句分数、LID审计、失败日志以及用于添加系统的脚本。

## Abstract
Text-to-speech (TTS) evaluation for low-resource non-Latin-script languages can fail when it relies on a single ASR round-trip word error rate (WER). A system may produce no audio, speak a neighbouring language, preserve target script text only in an ASR transcript, or sound unnatural to native listeners. We introduce INSV (Intelligibility, Naturalness, Script fidelity, and Verification), a reporting framework that separates these cases. This paper reports INSV-A, the automated screening subset: synthesis completion, ASR WER/CER, transcript Script Fidelity Rate, and audio language identification. Native MOS and phonetic annotation are specified but not claimed in this release. We instantiate INSV-A as PashtoTTS-Bench, a dated benchmark for Pashto TTS. The April-May 2026 run evaluates Edge GulNawaz, Edge Latifa, OmniVoice clone, OmniVoice auto, and an Urdu negative control on 200 FLEURS and 200 filtered Common Voice 24 prompts. Under the independent omniASR_CTC_300M_v2, OmniVoice auto has the lowest WER (24.1% FLEURS, 27.4% CV24), followed by Edge GulNawaz (32.8%, 39.5%), Edge Latifa (35.6%, 47.7%), and OmniVoice clone (45.4%, 34.8%). WER below the natural-speech baseline reflects clean synthetic audio and should not be read as better than native speech. Whisper Large V3 returns 0.0% Pashto labels on checked Pashto TTS audio, while MMS-LID-4017 and SpeechBrain VoxLingua107 separate Pashto outputs from the Urdu control. The release provides provider metadata, per-sentence scores, LID audits, failure logs, and scripts for adding systems.

### 1. 核心问题与整体含义（研究动机和背景）

- **问题**：低资源非拉丁文字语言的TTS评估，若仅依赖单一ASR往返WER（词错误率），会漏检三类关键失败：  
  - 系统不输出音频（F1：预合成拒绝）。  
  - 系统输出邻接语言（F2：语言替换）。  
  - 系统虽然输出目标语言但脚本崩溃（如普什图语特有字母被替换）。  
  此外，WER本身受ASR模型质量影响，且无法衡量自然度和语言正确性。

- **背景**：普什图语（Pashto）有6000–8000万使用者，使用波斯-阿拉伯文字，包含多个特有字母（如ټ、ډ、ڼ、ړ、ښ、ږ、ځ、څ），且缺乏公开发音词典和G2P规则。现有商用TTS仅微软提供普什图语语音，开源系统支持不明。缺乏公开的多系统TTS基准。

### 2. 方法论：核心思想、关键技术细节

- **INSV框架**：将TTS评估拆分为四个维度，避免单一WER失效：  
  - **I** (Intelligibility)：ASR往返WER/CER + 完美%  
  - **N** (Naturalness)：母语MOS（本文仅设计，不报告）  
  - **S** (Script fidelity)：转录脚本忠实度（SFR），衡量ASR假设中目标文字（如阿拉伯区块U+0600–U+06FF）的比率  
  - **V** (Verification)：语言ID（LID），多模型（MMS-LID-4017、SpeechBrain VoxLingua107）确认输出语言是否为目标语言

- **INSV-A自动化子集**（本文核心）：  
  - 评估合成完成率（是否产生非零音频）  
  - 基于独立ASR（omniASR_CTC_300M_v2）报告WER/CER  
  - 计算SFR  
  - 使用多模型LID输出Pashto-ID率  
  - 不要求人类MOS或语音标注

- **失败分类学（五类）**：  
  - F1：预合成拒绝（无输出）  
  - F2：语言替换（输出非目标语言）  
  - F3：音素崩塌（特有音素被邻接语言替代）  
  - F4：韵律异常（自然度低）  
  - F5：字形歧义（同形字符混淆，如U+06CC vs U+06D0）

### 6. 主要结论与发现

- **OmniVoice auto取得最低WER**：FLEURS 24.1%，CV24 27.4%（omniASR）。但该值低于自然语音基线（47.9%），作者强调这是因为合成音频声学洁净，非音质优于真人。  
- **系统排序一致**：OmniVoice auto < Edge GulNawaz < Edge Latifa < OmniVoice clone（在两个ASR下排序相似）。  
- **Whisper Large V3无法识别普什图语**：尽管词汇表中包含 `<|ps|>` 令牌，实际输出0%普什图语标签，表明训练数据中普什图语近乎缺失。MMS-LID和SpeechBrain可区分普什图语和乌尔都语。  
- **SFR未提供区分信号**：所有普什图语目标系统SFR=1.0，因均使用普什图语专用语音。  
- **失败模式**：乌尔都语对照为确认的F2；GulNawaz在三个句子上表现F5候选（U+06CC→U+06D0替换），需人工裁定。  
- **ASR循环性警告**：OmniVoice auto WER低于自然语音基线，说明不能将WER优于自然音直接解释为更好的TTS。

### 7. 优点

- **框架创新**：INSV将评估从单一WER扩展到四个独立维度，特别适合非拉丁文字语言。  
- **自动化筛查实用**：INSV-A能低成本检测F1、F2、SFR和LID问题，指明哪些系统需人类评估。  
- **失败分类学**：五类失败模式具有语言通用性和实例化，便于研究者诊断和比较。  
- **可复现性**：提供冻结提示集、元数据、脚本、哈希值，允许再生（商业音频不重分发，但可重新合成）。  
- **独立ASR优先**：采用omniASR避免作者训练模型与商业提供者的潜在关联。

### 8. 不足与局限

- **无人类MOS**：INSV-A不报告自然度，需后续母语MOS研究。  
- **ASR循环性风险**：虽采用独立ASR，但作者训练了pashto-asr-v3，且Edge TTS训练数据由作者创建，仍存在间接耦合。  
- **方言偏差未控制**：普什图语有主要方言变体（如Yousafzai、Waziri），测试集和ASR可能偏向特定方言，MOS时需报告对话者方言。  
- **SFR信息量有限**：当所有系统SFR=1.0时，无法区分内部音素或韵律质量。  
- **实验覆盖不足**：仅5个系统（4个有效），未包括Google Cloud TTS、OpenAI TTS、ElevenLabs等常见商用系统（因缺乏普什图语支持），Fish Speech未完成。  
- **仅2026年4-5月的一个时间戳**：系统行为可能随时间变化，需定期更新。  
- **未做跨语言验证**：INSV-A框架仅在普什图语上实证，其他语言需重新提供语种特定输入（ASR模型、文字范围、音素列表等）。
