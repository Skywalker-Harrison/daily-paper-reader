## 摘要
在这篇观点论文中，我们提出了MetaHOPE，一个错误严重性感知的标注框架，用于评估隐喻翻译。隐喻给机器翻译（MT）和自然语言理解与处理（NLU, NLP）带来了挑战，因为它具有语义复杂性、上下文依赖性和文化嵌入性等特征，可能导致NLP模型的歧义问题。为了探究最先进的NLP模型在翻译隐喻方面的表现，我们选择了三个代表系统，即GoogleMT、GPT5.4和Hunyuan-7b作为神经机器翻译（NMT）模型和大型语言模型（LLMs）。我们使用了两个人工标注的隐喻语料库，包括VUAMC和PSUCMC，用于英译中和中译英任务。我们使用的原始语料库是单语的，我们使用MetaHOPE框架进行了错误标注，并生成了人工后编辑的黄金参考译文，作为双语使用的新资源。我们相信，MetaHOPE隐喻翻译标注评估框架、平行语料库资源以及对最先进自动翻译模型的错误分析，对于隐喻翻译研究领域将具有实用价值并带来启示。我们在论文录用后公开分享我们的资源。

## Abstract
In this opinion paper, we propose MetaHOPE, an error severity-aware annotation framework for evaluating metaphor translations. Metaphors present challenges for machine translation (MT) and natural language understanding and processing (NLU, NLP), because it presents the features of semantic complexity, contextual dependency, and cultural embeddings that can lead to ambiguity issues for NLP models. To investigate how state-of-the-art NLP models perform on translating metaphors, we select three representative systems, i.e., GoogleMT, GPT5.4, and Hunyuan-7b as Neural MT (NMT) models and LLMs. We used two human-annotated metaphor corpora, including VUAMC and PSUCMC for English-to-Chinese and Chinese-to-English translation purposes. The original corpora we used are monolingual, where we carried out error annotation using the MetaHOPE framework, and also produced the human post-edited gold reference for bilingual use as a new resource. We believe the MetaHOPE evaluation framework for metaphor translation annotation, the parallel corpora resources, and the error analysis on SOTA automatic translation models can be useful and shed some light for the field of metaphor translation study. We share our resources publicly upon paper acceptance.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将翻译质量评估中的HOPE框架改编为隐喻专用版本（MetaHOPE），保持轻量化设计，同时聚焦隐喻相关的典型错误类型。
- **关键技术细节**：
  - **错误类别**（共5类）：
    1. **Impact (IMP)**：过度字面直译或结构改变影响强调（如主动变被动）。
    2. **Required Adaptation Missing (RAM)**：缺少必要的文化或习语适应（如未将“black sheep”译为“害群之马”）。
    3. **Mistranslation (MIS)**：意义错配，错误理解隐喻含义（如将“on its last legs”直译为“在最后几条腿上”）。
    4. **Style (STL)**：丧失隐喻效果、意象或情感语气（如弱化“lashed out”为“表示不满”）。
    5. **Proof-reading Error (PRF)**：表达不自然但未改变含义（如直译“动量”而非“势头”）。
  - **严重性评分**：采用5级评分（2, 4, 6, 8, 10），代替原HOPE的指数稀疏评分（1,2,4,8,16），以降低标注者间分歧。
  - **评估流程**：
    - ① 从单语隐喻语料库（VUAMC/PSUCMC）提取文本并格式化。
    - ② 对完整文档进行上下文感知的机器翻译（文档级翻译，再切分回句子）。
    - ③ 手动对齐源语言隐喻词（MRW）与目标语言对应部分。
    - ④ 进行人工标注（MetaHOPE评分）并生成后编辑参考译文。
- **排除类别**：原HOPE中的Terminology（TRM）、Proper Name（PRN）、Ungrammatical（UGR）因不直接反映隐喻处理而被移除。

## 6. 论文的主要结论与发现

- **隐喻错误是主要错误来源**：隐喻相关错误（EPM）占全句子错误（EPS）的比例分别为：GoogleMT 91.7%、GPT-5.4 93.8%、Hunyuan-LLM-7B 61.8%。说明当前系统在隐喻翻译上仍存在显著瓶颈。
- **系统间差异**：
  - GoogleMT和GPT-5.4偏向保守的直译，保持源语言结构。
  - Hunyuan-LLM-7B更灵活，有时能产出更地道的译文（如用“自主管理”代替字面的“信托地位”），但也会出现事实性错误或幻觉（如添加原文没有的信息）。
- **标注者一致性**：Pearson r分别为GoogleMT 0.536，GPT-5.4 0.726，Hunyuan 0.333。GPT-5.4的产出最稳定，Hunyuan产出更难解释导致标注者分歧大。
- **定性发现**：包括术语翻译不一致、词级对齐困难、中文近义词选择（如“做出”与“作出”）等。

## 7. 优点

- **方法亮点**：
